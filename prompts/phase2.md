# Phase 2: 지능형 검색 및 내비게이션 고도화

---

## 1. 프로젝트 개요
- **목표**: 250여 개의 방대한 JJSTAR 메뉴 구조 내에서 사용자가 원하는 가이드를 즉시 찾고, 현재 위치를 길을 잃지 않고 탐색할 수 있는 UX 구현
- **핵심 작업**: 초성 검색 엔진 구현, 검색어 하이라이팅, 사이드바 Active 상태 동기화, 브레드크럼(Breadcrumb) 자동화
- **기술 제약**: 외부 라이브러리(jQuery 등) 없이 순수 Vanilla JS로만 로직 구현

---

## 2. 세부 수행 단계

### [Step 1] 초성 검색 및 검색어 하이라이팅 구현
`assets/js/search.js` (또는 메인 스크립트의 검색 로직)를 고도화하세요.
1.  **초성 추출 알고리즘**: 한글 입력 시 초성(ㄱㄴㄷ...)을 분리하는 함수를 작성하여, 'ㅅㅈㄷㅎ' 입력 시 '사제동행교수선택'이 검색되도록 구현합니다.
2.  **실시간 필터링**: `window.JJSTAR_MENU` 데이터와 사용자의 입력을 실시간 매칭하여 일치하는 항목만 사이드바에 즉시 노출합니다.
3.  **검색어 강조(Highlight)**: 결과 리스트에서 사용자가 입력한 키워드와 일치하는 부분에 `<mark>` 태그를 입혀 시각적으로 강조합니다.

### [Step 2] 사이드바 내비게이션 UX 고도화
1.  **Active 상태 관리**: 현재 로드된 페이지(`path`)에 해당하는 사이드바 메뉴 항목에 `.is-active` 클래스를 추가하고, 해당 항목이 속한 부모 카테고리가 자동으로 펼쳐지도록(Accordion Open) 구현합니다.
2.  **자동 스크롤**: 메뉴 리스트가 길 경우, 선택된 항목이 사이드바 화면 중앙에 오도록 `element.scrollIntoView({ block: 'center' })` 로직을 추가합니다.
3.  **결과 카운팅**: 검색창 하단에 '검색 결과: N건'과 같은 상태 메시지를 표시하여 사용자에게 피드백을 제공합니다.

### [Step 3] 동적 브레드크럼(Breadcrumb) 자동 생성
가이드 본문 영역 상단에 현재 문서의 계층 위치를 나타내는 내비게이션 바를 생성합니다.
1.  **계층 역추적**: 현재 표시 중인 소분류의 `code`를 기준으로 `menu.js`를 역추적하여 `대분류 > 중분류 > 소분류` 형태의 경로 텍스트를 생성합니다.
2.  **헤더 동기화**: 문서가 교체될 때마다 브레드크럼과 문서 제목(H1)이 자동으로 업데이트되도록 JS 함수를 연결합니다.

## 3. 검증 포인트 (Checklist)

- [ ] **초성 검색**: 'ㅎㅁㅎㄱ' 검색 시 '희망학과 선택' 메뉴가 정상적으로 필터링되는가?
- [ ] **성능**: 200개 이상의 메뉴 항목을 실시간 필터링할 때 화면 끊김이나 지연이 없는가?
- [ ] **네비게이션**: '이전/다음' 버튼으로 이동할 때 사이드바의 하이라이트 위치가 실시간으로 변경되는가?
- [ ] **가독성**: 검색 결과 내의 하이라이트(mark) 표시가 배경색과 대비되어 잘 보이는가?
