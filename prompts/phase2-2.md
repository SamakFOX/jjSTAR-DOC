# Phase 2-2: 사이드바 아코디언 로직 및 검색 필터링 최적화

---

## 1. 프로젝트 개요
- **목표**: 검색 결과의 가독성을 높이기 위해 매칭되지 않은 그룹은 접힌 상태로 유지하고, 사이드바의 접고 펼치기(Accordion) 동작을 사용자 친화적으로 개선
- **핵심 작업**: 검색 매칭 기반 자동 개폐 로직, 중분류 접기 기능 추가, 배타적 펼치기(하나만 열기) 로직 복구
- **기술 제약**: Vanilla JS 사용, 기존 `menu.js` 데이터 구조 유지

---

## 2. 세부 수행 단계

### [Step 1] 검색 매칭 기반 지능형 개폐 로직
검색어 입력 시 모든 메뉴가 펼쳐지는 대신, 검색어와 직접 일치하는 항목만 강조하고 나머지는 관리합니다.
1.  **직접 매칭 우선**: 소분류(최하위) 제목에 검색어가 포함된 경우에만 해당 부모(중분류, 대분류)를 펼칩니다.
2.  **간접 매칭 처리**: 대분류나 중분류 이름에 검색어가 포함되어 검색 결과에 포함된 것이라면, 그 하위 소분류들은 기본적으로 **접힌 상태(Collapsed)**로 노출하여 화면 낭비를 줄입니다.
3.  **검색 초기화**: 검색어를 모두 지우면 모든 메뉴를 다시 초기 상태(닫힘)로 되돌립니다.

### [Step 2] 아코디언(접고 펼치기) 기능 강화
1.  **중분류 접기 추가**: 현재 대분류만 접기가 가능하던 로직을 수정하여, 중분류 제목 클릭 시에도 하위 소분류 리스트가 토글(접고 펼치기)되도록 기능을 추가합니다.
2.  **배타적 펼치기(Single Expand) 복구**: 
    - 특정 대분류나 중분류를 클릭하여 펼치면, 이미 열려있던 다른 동급의 메뉴들은 자동으로 접히도록 구현합니다.
    - 이를 통해 사용자가 한 번에 하나의 카테고리에만 집중할 수 있도록 사이드바 길이를 조절합니다.

### [Step 3] 검색 결과 내 시각적 구분
1.  **매칭 강조**: 검색어와 직접 일치하는 텍스트는 `<mark>` 태그나 별도의 CSS 클래스를 통해 시각적으로 도드라지게 표시합니다.
2.  **비매칭 그룹 저채도화**: 검색 결과에는 포함되었으나 검색어가 직접 들어있지 않은 상위 카테고리는 약간 흐리게(Opacity 등) 처리하여 우선순위를 구분합니다.

## 3. 검증 포인트 (Checklist)

- [ ] **필터링**: '학사' 검색 시, 소분류에 '학사'가 없는 '교육과정' 등은 제목만 나오고 하위 목록은 접혀 있는가?
- [ ] **토글**: 중분류 제목을 클릭했을 때 하위 항목들이 정상적으로 접히고 펼쳐지는가?
- [ ] **상태 유지**: 새로운 메뉴를 열었을 때 기존에 열려있던 다른 메뉴들이 자동으로 닫히는가?
- [ ] **검색어 강조**: 입력한 키워드가 제목 내에서 정확히 하이라이트 되는가?
