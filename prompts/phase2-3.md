# Phase 2-3: 대/중분류 통합 접기 로직 및 검색 유연성 강화

---

## 1. 프로젝트 개요
- **목표**: 설명 페이지가 없는 '대분류'의 클릭 피드백을 강화하고, 검색 중에도 사용자가 자유롭게 메뉴를 접고 펼칠 수 있도록 아코디언 로직 완성
- **핵심 작업**: 대분류/중분류 통합 토글(Toggle) 로직 구현, 검색 시 대분류 개방 유지, 배타적 펼치기 기능 정교화
- **기술 제약**: Pure Vanilla JS, 기존 `menu.js` 데이터 구조 활용

---

## 2. 세부 수행 단계

### [Step 1] 대분류(Category) 클릭 이벤트 및 피드백 강화
1.  **무조건 토글**: 대분류는 별도의 설명 HTML이 없으므로, 클릭 시 페이지 이동 대신 **하위 중분류 리스트를 접고 펼치는 기능**만 수행하도록 고정합니다.
2.  **검색 시 예외 처리**: '정보' 검색 시 '정보공유' 대분류에 직접 매칭되는 소분류가 없더라도, 사용자가 대분류 제목을 클릭하면 **언제든지 하위 메뉴를 열어서 확인**할 수 있도록 `display: none` 제어 로직을 수정합니다.

### [Step 2] 중분류(Mid) 접기 기능 및 페이지 로드 통합
1.  **중분류 토글 구현**: 중분류 제목 클릭 시 다음 두 가지 동작이 동시에 일어나야 합니다.
    - 하위 소분류(Small) 리스트 접고 펼치기 (Toggle)
    - 해당 중분류의 설명 HTML 페이지 로드 (Fetch)
2.  **아이콘 상태 변경**: 접혀있을 때와 펼쳐졌을 때의 화살표 방향(또는 아이콘)이 시각적으로 변하도록 CSS와 연동합니다.

### [Step 3] 배타적 펼치기(One-at-a-time) 및 상태 관리
1.  **동일 레벨 간섭**: 
    - 새로운 대분류를 열면 기존에 열려있던 대분류는 닫힙니다.
    - 대분류 내에서 새로운 중분류를 열면 해당 대분류 내의 다른 중분류는 닫힙니다.
2.  **검색 매칭 시 자동 개방**: 소분류에 검색어가 포함되어 직접 매칭된 경우에만 해당 부모들을 '자동'으로 펼치되, 사용자가 수동으로 다시 접는 것을 방해하지 않도록 설계합니다.

---

## 3. 검증 포인트 (Checklist)

- [ ] **대분류 반응**: 검색 결과와 상관없이 대분류 제목 클릭 시 하위 메뉴가 부드럽게 열리고 닫히는가?
- [ ] **중분류 접기**: 중분류를 클릭했을 때 페이지는 로드되면서 하위 메뉴 리스트만 접었다 펼쳤다 할 수 있는가?
- [ ] **배타적 동작**: 'star 학생지원'을 연 상태에서 '장학안내'를 클릭하면 'star 학생지원'이 자동으로 접히는가?
- [ ] **검색 후 탐색**: '정보' 검색 후 닫혀있는 '정보공유' 대분류를 클릭했을 때 내부 중분류들이 정상적으로 노출되는가?
