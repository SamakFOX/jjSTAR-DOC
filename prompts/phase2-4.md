# Phase 2-4: 검색 모드와 일반 모드의 분리 및 탐색 유연성 확보

---

## 1. 프로젝트 개요
- **목표**: 검색 결과에서도 메뉴의 전체 구조를 유지하여 탐색의 연속성을 확보하고, 검색 모드와 일반 탐색 모드의 아코디언 동작을 차별화함
- **핵심 작업**: 검색 모드 시 '배타적 펼치기' 비활성화, 직접 매칭되지 않은 하위 메뉴의 시각적 저채도화, 대분류 강제 개방 로직 수정
- **기술 제약**: Pure Vanilla JS, 기존 CSS 시스템 활용

---

## 2. 세부 수행 단계

### [Step 1] 검색 모드 전용 아코디언 로직 적용
1.  **배타적 펼치기 예외 처리**: 
    - 검색어가 입력된 상태(Search Mode)에서는 하나를 열었을 때 다른 메뉴가 닫히는 로직을 **일시 중지**합니다. 
    - 사용자가 필터링된 여러 결과를 동시에 펼쳐놓고 비교할 수 있도록 합니다.
2.  **검색 시 대분류 개방 유지**: '정보' 검색 시 '정보공유'처럼 대분류만 매칭된 경우에도 클릭 시 하위 중분류들이 정상적으로 노출되도록 `display: none` 제어를 해제합니다.

### [Step 2] 하위 메뉴 필터링 방식 개선 (숨김 대신 흐리게)
하위 메뉴가 필터링 조건에 맞지 않아 아예 사라지는 문제를 해결합니다.
1.  **시각적 구분 (Opacity)**: 
    - 검색어와 직접 일치하는 소분류는 하이라이트(`<mark>`) 처리합니다.
    - 검색어와 일치하지 않는 소분류는 아예 삭제하는 대신, `opacity: 0.4` 정도로 흐리게 표시하여 메뉴의 전체 구조를 유지합니다.
2.  **탐색 가능성 유지**: 흐리게 표시된 메뉴라도 사용자가 원한다면 클릭하여 해당 가이드 페이지로 이동할 수 있도록 합니다.

### [Step 3] 일반 모드(기본 상태) 로직 복구 및 정교화
검색어가 없을 때는 이전의 편리한 아코디언 기능을 유지합니다.
1.  **자동 복구**: 검색창을 비우면 모든 메뉴는 다시 '접힘' 상태로 돌아가며, '하나만 열기(배타적 펼치기)' 로직이 다시 활성화됩니다.
2.  **대분류 피드백**: 설명 페이지가 없는 대분류는 클릭 시 페이지 로드 없이 하위 메뉴 토글 기능만 수행하도록 로직을 고정합니다.

---

## 3. 검증 포인트 (Checklist)

- [ ] **검색 중 배타적 동작**: 검색 결과 창에서 여러 대분류를 열었을 때 이전에 연 메뉴가 닫히지 않고 유지되는가?
- [ ] **하위 메뉴 노출**: 필터링에 걸리지 않은 소분류들이 완전히 사라지지 않고 흐릿하게라도 보이는가?
- [ ] **대분류 클릭**: 검색 결과에 자식 요소가 없더라도 '정보공유' 같은 대분류 클릭 시 내부 중분류 리스트가 열리는가?
- [ ] **일반 모드 전환**: 검색어를 지우면 다시 깔끔하게 메뉴들이 접히고 배타적 펼치기 로직이 작동하는가?
