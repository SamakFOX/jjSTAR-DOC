# Phase 2-6: 검색 필터링 최적화 및 사이드바 안정화 (최종)

---

## 1. 프로젝트 개요
- **목표**: 검색 시 매칭되지 않는 항목을 완벽히 제거하여 검색 결과의 가독성을 높이고, 일반 탐색 시의 안정적인 아코디언 로직을 유지함
- **핵심 작업**: 검색 매칭 항목 외 완전 숨김(display: none), 검색 시 대분류/중분류 자동 개방, 일반 모드 시 배타적 펼치기 복구
- **기술 제약**: Pure Vanilla JS, 불필요한 opacity나 복잡한 예외 로직 제거

---

## 2. 세부 수행 단계

### [Step 1] 검색 모드: 정교한 데이터 필터링 (Hide non-matching)
1.  **완전 숨김 로직**: 검색어가 입력되면, 해당 검색어를 포함하지 않는 소분류(Small) 항목들은 화면에서 완전히 숨깁니다. (`display: none`)
2.  **부모 요소 제어**: 
    - 하위 항목 중 하나라도 매칭되는 것이 있다면 그 부모(대분류, 중분류)는 화면에 노출하고 자동으로 펼칩니다.
    - 하위 항목 중 매칭되는 것이 하나도 없는 대/중분류는 화면에서 숨깁니다.
3.  **대분류 검색 예외**: 대분류 이름 자체에 검색어가 포함된 경우(예: '정보'), 직접 매칭된 소분류가 없더라도 해당 대분류는 노출하며, 클릭 시 하위 메뉴를 사용자가 직접 열어볼 수 있도록 허용합니다.

### [Step 2] 일반 모드: 배타적 아코디언 로직 복구
1.  **하나만 열기(Single Expand)**: 
    - 검색어가 없을 때는 대분류 하나를 열면 다른 대분류가 닫히는 기본 동작을 보장합니다.
    - 중분류 역시 동일한 부모 내에서 하나만 열리도록 설정하여 사이드바 길이를 최적화합니다.
2.  **상태 초기화**: 검색창의 내용을 지우면 모든 메뉴는 다시 '접힘' 상태로 돌아가며 필터링이 해제됩니다.

### [Step 3] 클릭 피드백 및 페이지 로드 안정화
1.  **대분류 클릭**: 설명 페이지가 없는 대분류는 클릭 시 페이지 로드 없이 하위 메뉴를 토글(접고 펼치기)하는 동작만 수행합니다.
2.  **중분류 클릭**: 중분류 클릭 시 해당 가이드 HTML을 불러옴(Fetch)과 동시에 하위 소분류 리스트가 있다면 토글합니다.
3.  **검색어 강조**: 검색 결과로 나타난 텍스트 내의 키워드는 `<mark>` 태그로 강조하여 왜 검색되었는지 직관적으로 알게 합니다.

## 3. 검증 포인트 (Checklist)

- [ ] **필터링**: '학사' 검색 시 관련 없는 '교육과정' 등의 소분류가 화면에서 완전히 사라지는가?
- [ ] **자동 개방**: 검색 결과가 있는 카테고리는 별도 클릭 없이도 시원하게 펼쳐져서 보이는가?
- [ ] **배타적 동작**: 일반 모드에서 다른 대분류를 누르면 기존에 열려있던 메뉴가 즉시 닫히는가?
- [ ] **대분류 노출**: 자식 중에 매칭이 없더라도 대분류 이름이 검색어와 일치하면 화면에 나타나는가?
