# Phase 2-5: 사이드바 로직 안정화 및 기본 아코디언 복구

---

## 1. 프로젝트 개요
- **목표**: 복잡한 검색 예외 로직을 제거하고, 직관적이고 빠른 기본 사이드바 내비게이션 및 검색 시스템으로 복구
- **핵심 작업**: 배타적 펼치기(하나만 열기) 로직 복원, 검색 시 일치 항목만 노출하는 기본 필터링 회귀, 대분류 클릭 시 토글 기능 보장
- **기술 제약**: Pure Vanilla JS 사용, 불필요한 조건문 제거로 코드 경량화

---

## 2. 세부 수행 단계

### [Step 1] 기본 아코디언 및 배타적 펼치기 복구
1.  **배타적 펼치기(Single Expand)**: 
    - 검색 여부와 상관없이, 대분류를 열면 다른 대분류가 닫히고, 중분류를 열면 같은 부모 아래의 다른 중분류가 닫히는 로직을 최우선으로 적용합니다.
    - 이를 통해 사이드바가 항상 일정한 길이를 유지하도록 합니다.
2.  **검색 필터링 원복**: 
    - 검색어와 일치하지 않는 항목(소분류)은 `display: none` 처리하여 화면에서 완전히 숨깁니다. 흐리게 처리하는 로직은 삭제합니다.

### [Step 2] 대분류(Category) 클릭 및 개방성 보장
1.  **대분류 반응성**: 
    - 대분류는 설명 페이지가 없으므로 클릭 시 페이지 이동을 시도하지 않고 오직 **하위 메뉴의 접고 펼치기(Toggle)** 기능만 수행합니다.
    - 검색 결과가 대분류에만 걸려 있더라도, 클릭하면 해당 대분류가 펼쳐지며 내부 중분류 목록을 볼 수 있어야 합니다.
2.  **자동 개방**: 검색어와 직접 일치하는 소분류가 있는 경우, 그 부모인 대분류와 중분류는 자동으로 펼쳐진 상태로 렌더링합니다.

### [Step 3] 코드 정리 및 버그 수정
1.  **중분류 접기 확인**: 중분류 클릭 시 해당 페이지를 로드함과 동시에 하위 소분류 목록이 정상적으로 토글되는지 점검합니다.
2.  **상태 초기화**: 검색창을 비우면 모든 메뉴가 초기 상태(모두 접힘)로 깔끔하게 돌아가도록 로직을 정돈합니다.

---

## 3. 검증 포인트 (Checklist)

- [ ] **배타적 동작**: 새로운 카테고리를 클릭했을 때 이전에 열려있던 카테고리가 자동으로 잘 접히는가?
- [ ] **검색 결과**: 검색어와 상관없는 메뉴들이 숨겨져서 검색 결과가 한눈에 들어오는가?
- [ ] **대분류 클릭**: 검색 결과 창에서도 대분류를 클릭했을 때 반응(열기/닫기)이 확실한가?
- [ ] **안정성**: 메뉴를 여러 번 빠르게 클릭하거나 검색어를 지웠을 때 레이아웃이 깨지지 않는가?
