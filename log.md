## [2026-03-09] Phase 2: 지능형 검색 및 내비게이션 고도화

### 작업 요약
방대한 메뉴 구조에서 원하는 정보를 즉시 찾고 현재 위치를 직관적으로 파악할 수 있도록 검색 엔진 및 내비게이션 UX를 대폭 강화했습니다.

### 상세 변경 내역

#### 1. 지능형 초성 검색 엔진 구현 (`assets/js/search.js`)
- **초성 검색**: 한글 초성(ㄱㄴㄷ...)만으로 메뉴를 검색할 수 있는 알고리즘을 도입하여 접근성을 높였습니다. (예: 'ㅅㅈㄷㅎ' -> '사제동행교수선택')
- **실시간 필터링**: 입력 시 즉시 `JJSTAR_MENU` 데이터를 필터링하여 사이드바를 업데이트합니다.
- **키워드 하이라이팅**: 검색어와 일치하는 텍스트에 `<mark>` 태그를 적용하여 시각적으로 강조했습니다.

#### 2. 사이드바 내비게이션 UX 강화 (`index.html`, `style.css`)
- **Active 상태 동기화**: 현재 읽고 있는 문서에 해당하는 메뉴 항목에 `.is-active` 스타일(배경색 및 강조선)을 실시간으로 적용합니다.
- **자동 아코디언**: 문서 로드 시 해당 항목이 속한 상위 카테고리 및 중분류가 자동으로 펼쳐지도록 개선했습니다.
- **Auto-Scroll**: 선택된 메뉴 항목이 사이드바에서 보이지 않을 경우, 자동으로 화면 중앙으로 스크롤(`scrollIntoView`)되도록 구현했습니다.
- **검색 결과 카운팅**: 검색창 하단에 현재 필터링된 메뉴의 개수를 실시간으로 표시합니다.

#### 3. 동적 경로 추적 및 헤더 동기화
- **Dynamic Breadcrumb**: 메뉴 데이터를 역추적하여 현재 위치를 `대분류 > 중분류 > 소분류` 형태로 정확하게 표시합니다.
- **제목 자동 업데이트**: 문서 로드 시 해당 메뉴의 제목이 본문 H1 영역에 자동으로 반영되도록 구현했습니다.

## [2026-03-09] Phase 2-2: 사이드바 아코디언 로직 및 검색 필터링 최적화

### 작업 요약
방대한 메뉴 구조에서 사용자가 길을 잃지 않도록 사이드바의 접고 펼치기(Accordion) 동작을 더욱 지능적으로 개선하고, 검색 결과의 시각적 위계를 강화했습니다.

### 상세 변경 내역

#### 1. 검색 매칭 기반 지능형 개폐 로직 (`search.js`, `index.html`)
- **소분류 매칭 우선 펼침**: 검색어가 소분류(최하위 메뉴)와 직접 일치하는 경우에만 부모 카테고리를 자동으로 펼칩니다.
- **간접 매칭 항목 접기**: 대분류나 중분류 제목만 검색어와 일치하는 경우, 하위 메뉴는 기본적으로 접힌 상태로 노출하여 불필요한 화면 낭비를 줄였습니다.
- **비매칭 그룹 저채도화**: 검색 결과에는 포함되었으나 키워드가 직접 포함되지 않은 상위 항목들은 흐리게(Opacity 적용) 표시하여 시각적 우선순위를 구분했습니다.

#### 2. 아코디언(접고 펼치기) UX 고도화
- **중분류 토글 기능**: 기존 대분류만 가능하던 접기 기능을 중분류 제목 클릭 시에도 동작하도록 확장했습니다.
- **배타적 펼침(Single Expand)**: 특정 카테고리나 중분류를 열면 동일 레벨의 다른 메뉴들이 자동으로 닫히도록 하여, 사용자가 현재 선택한 영역에만 집중할 수 있게 했습니다.
- **이벤트 전파 방지**: 메뉴 클릭 시 의도치 않은 중복 동작이 발생하지 않도록 `stopPropagation` 로직을 적용했습니다.

## [2026-03-09] Phase 2-3: 대/중분류 통합 접기 로직 및 검색 유연성 강화

### 작업 요약
설명 페이지가 없는 대분류의 탐색 경험을 개선하고, 검색 중에도 사용자가 자유롭게 메뉴를 제어할 수 있도록 아코디언 로직을 완성했습니다.

### 상세 변경 내역

#### 1. 대/중분류 통합 토글(Toggle) 로직 완성
- **대분류 클릭 피드백**: 별도의 페이지가 없는 대분류 클릭 시, 하위 중분류 리스트를 접고 펼치는 기능만 수행하도록 직관적으로 개선했습니다.
- **중분류 접기 기능 통합**: 중분류 클릭 시 페이지 로드(Fetch)와 하위 소분류 리스트의 토글이 동시에 발생하도록 통합했습니다.
- **상태 표시 강화**: 접힘/펼침 상태에 따라 사이드바의 인디케이터(+/-) 및 아이콘 상태가 정확하게 동기화됩니다.

#### 2. 지능형 배타적 펼치기(Exclusive Expand) 구현
- **계층형 단일 선택**: 새로운 대분류를 열면 다른 대분류가 닫히고, 대분류 내에서 새로운 중분류를 열면 해당 대분류 내 다른 중분류들이 자동으로 접히도록 정교화했습니다.
- **검색 유연성 확보**: 검색어 입력 시 소분류 매칭이 있는 항목은 자동으로 펼쳐지지만, 사용자가 수동으로 다시 접거나 펼치는 동작을 방해하지 않도록 상태 관리 로직을 개선했습니다.

#### 3. 검색 시 탐색 자유도 보장
- **검색 개방 유지**: 검색 결과에 포함된 대분류라면, 직접 매칭되는 항목이 없더라도 클릭을 통해 하부 구조를 언제든지 탐색할 수 있도록 `display` 로직을 최적화했습니다.

## [2026-03-10] Phase 2-6: 검색 필터링 최적화 및 사이드바 안정화 (최종)

### 작업 요약
검색 시 매칭되지 않는 항목을 완벽히 제거하여 검색 결과의 가독성을 극대화하고, 모드별(검색/일반) 최적화된 아코디언 동작을 구현하여 탐색 안정성을 최종적으로 확보했습니다.

### 상세 변경 내역

#### 1. 정교한 데이터 필터링 (Hide non-matching)
- **완전 숨김 로직**: 검색어와 일치하지 않는 소분류(Small) 항목들을 목록에서 완전히 제외하여 노이즈 없는 검색 결과를 제공합니다.
- **계층적 노출 조건**: 
  - 하위 소분류 중 하나라도 매칭되거나, 중분류 제목이 매칭될 경우에만 해당 중분류를 노출합니다.
  - 대분류 제목 자체가 검색어와 일치하는 경우(예: '학사'), 하위 모든 소분류가 매칭되지 않더라도 대분류와 중분류 구조를 노출하여 사용자가 수동으로 탐색할 수 있도록 허용합니다.

#### 2. 모드별 아코디언 로직 이원화
- **일반 모드 (Normal Mode)**: '하나만 열기(Single Expand)' 배타적 펼침 로직을 복구하여, 대분류나 중분류를 열면 동일 레벨의 다른 메뉴가 자동으로 접히도록 최적화했습니다.
- **검색 모드 (Search Mode)**: 매칭된 여러 결과를 한눈에 비교할 수 있도록 배타적 펼침 로직을 일시 중지하고, 다중 개방 상태를 유지합니다.
- **자동 개방(Auto-Expand)**: 하위 검색 결과가 존재하는 부모 메뉴는 별도 클릭 없이도 자동으로 펼쳐진 상태로 렌더링됩니다.

#### 3. UX 인터랙션 및 상태 관리 최적화
- **대분류 토글 전용화**: 설명 페이지가 없는 대분류 클릭 시 페이지 이동 없이 하위 리스트의 접고 펼치기 기능만 수행합니다.
- **검색어 강조(Highlighting)**: 검색된 텍스트 내 키워드에 `<mark>` 태그를 적용하여 검색 결과 시인성을 높였습니다.
- **검색 결과 카운트**: 화면에 실시간으로 필터링되어 노출된 유효 항목의 개수를 정확하게 표시합니다.
